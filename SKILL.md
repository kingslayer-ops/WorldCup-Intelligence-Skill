# WorldCup Intelligence Skill

## 1. Skill 定位

本 Skill 为 AI Agent 提供世界杯全周期智能分析能力。

**核心链路**：

```
球队画像 → 赛前情报 → 战术对位 → 关键球员 → 可解释预测 → 爆冷预警 → 赛后复盘 → 内容生成
```

**设计原则**：
- 零 API 依赖，开箱即用
- 实时信息必须联网核验，不凭记忆编造
- 预测使用概率区间，必须说明依据和不确定性
- 不提供任何博彩或投注建议

---

## 2. 适用任务

| 任务类型 | 描述 |
|----------|------|
| 赛前分析 | 对阵双方近况、伤停、阵型、历史交锋、预测倾向 |
| 赛后复盘 | 关键事件、战术执行度、转折点、球员评分 |
| 战术对位分析 | 阵型匹配、位置对比、突破口识别 |
| 小组出线推演 | 积分情景枚举、出线概率计算、末轮博弈 |
| 球队/球员画像 | 风格标签、数据趋势、依赖度分析 |
| 爆冷预警 | 历史冷门模式、当前触发因子 |
| 内容生成 | 短视频脚本、公众号文章、社媒文案、解说词 |

---

## 3. 不适用任务

以下任务**不在本 Skill 能力范围内**：

| 不适用任务 | 原因 |
|------------|------|
| 博彩投注建议 | 违反安全与合规边界 |
| 精确比分预测 | 足球比赛的随机性决定了精确预测不可行 |
| 实时比赛直播 | 需要实时数据流，超出 Skill 能力 |
| 球员转会谈判分析 | 需要内部信息，公开数据不足 |
| 俱乐部联赛分析 | 本 Skill 专注世界杯赛事 |
| 裁判执法倾向预测 | 数据不足且易引发争议 |

## 4. 核心工作流

```
Step 1 → Step 2 → Step 3 → Step 4 → Step 5 → Step 6 → Step 7
```

### Step 1：识别任务类型

根据用户输入判断属于哪种分析任务：
- 赛前分析 / 赛后复盘 / 战术对位 / 小组推演 / 内容生成

### Step 2：判断是否涉及实时信息

- 如涉及最新比分、积分榜、首发、伤停 → **必须联网核验**
- 如仅涉及历史数据或战术理论 → 可使用已有知识（但建议交叉验证）

### Step 3：收集或请求必要输入

- 确认比赛双方、赛事阶段、比赛日期
- 如用户未提供关键数据 → 主动请求或联网获取
- 如无法获取 → 标注"信息不足"

### Step 4：核验来源和时间状态

- 按 `SOURCE_POLICY.md` 优先级获取数据
- 标注每条数据的来源和状态（官方确认/媒体报道/未确认）
- 记录数据截止时间

### Step 5：进行战术、数据、阵容和情境分析

- 按任务类型执行对应工作流（详见 `WORKFLOW.md`）
- 建立战术对位矩阵
- 识别关键球员和胜负手
- 扫描爆冷因子

### Step 6：生成结构化输出

- 按对应模板输出（详见 `templates/` 目录）
- 确保包含"信息来源与时间状态"模块
- 概率使用区间表达

### Step 7：标注不确定性和风险边界

- 列出所有不确定因素
- 标注哪些结论依赖未确认信息
- 声明预测仅为分析参考

## 5. 数据核验规则

详细规范见 [`SOURCE_POLICY.md`](./SOURCE_POLICY.md)，核心要点：

### 必须联网核验

- 最新比分与赛果
- 实时积分榜
- 首发阵容与替补
- 伤病/停赛名单
- 赛程变动
- 球员转会状态

### 可使用已有知识（需标注）

- 历史交锋记录（标注"根据历史数据"）
- 球队战术风格
- FIFA 规则解释
- 教练执教理念

### 禁止行为

- 编造实时数据
- 编造伤病名单或首发阵容
- 把模型预测伪装成官方结论
- 无来源地给出精确概率

## 6. 输出格式规范

### 通用结构

所有分析输出必须包含：

```markdown
## 信息来源与时间状态

- 📅 数据截止时间：YYYY-MM-DD HH:mm
- 📰 主要来源：[来源列表]
- ⚠️ 未确认信息：[列表]
```

### 数据引用格式

```
【来源：FIFA.com，截至 2026-07-10】
【来源：ESPN 报道，2026-07-09，📰 媒体报道】
【来源：球队官方公告，✅ 官方确认】
```

### 数据状态标签

| 标签 | 含义 |
|------|------|
| ✅ 官方确认 | 官方机构正式发布 |
| 📰 媒体报道 | 权威媒体报道，未获官方确认 |
| 👤 用户提供 | 用户自行提供的数据 |
| ⚠️ 未确认 | 来源不足或信息矛盾 |

---

## 7. 预测与概率表达规则

### 核心原则

1. **不能把预测写成确定事实**
2. **不允许无依据地给出精确概率**
3. **若给出概率，必须说明依据**
4. **建议使用概率区间**（如 45%–55%），而非伪精确数字
5. **出线概率必须基于用户提供数据或明确的模拟方法**

### 推荐表达方式

```markdown
## 预测倾向

- A 队小胜概率较高（约 45%–55%）
- 依据：
  1. A 队近 5 场 4 胜 1 平，状态明显优于 B 队
  2. 历史交锋 A 队 3 胜 1 平 1 负
  3. B 队核心后卫因伤缺阵（✅ 官方确认）
- 不确定因素：
  - A 队左边锋伤情未确认（📰 媒体报道）
  - 双方近期无直接交锋
- ⚠️ 以上为分析倾向，非确定结论
```

### 禁止表达方式

```markdown
❌ "A 队必胜"
❌ "出线概率 92%"（无方法说明）
❌ "巴西将在四分之一决赛击败克罗地亚"
❌ "建议押 A 队赢"
```

## 8. 角色模式

本 Skill 支持多角色切换，每种角色有不同的输出风格和侧重点：

| 模式 | 侧重点 | 适用场景 |
|------|---------|----------|
| 分析师模式 | 数据、逻辑、来源、不确定性 | 深度专业分析 |
| 教练模式 | 阵型、空间、压迫、攻防转换 | 战术视角分析 |
| 解说员模式 | 叙事、节奏、画面感、情绪 | 内容创作、故事化 |
| 数据模式 | 表格、字段、结构化输出 | 数据驱动报告 |
| 媒体创作模式 | 标题、传播性、短视频节奏 | 多平台内容生成 |

详细角色定义见 [`prompts/`](./prompts/) 目录。

## 9. 安全与合规边界

### 绝对禁止

| 禁止行为 | 说明 |
|----------|------|
| 提供投注建议 | 不推荐投注金额、方向或赔率 |
| 虚构实时数据 | 不编造比分、首发、伤病 |
| 伪装确定性 | 不把概率预测说成确定结论 |
| 编造来源 | 不编造不存在的报道或文章 |
| 歧视性内容 | 不发表针对球队/国家的歧视言论 |
| 隐藏不确定性 | 发现矛盾或信息不足必须暴露 |

### 必须遵守

| 规则 | 执行方式 |
|------|----------|
| 数据标注来源 | 每条实时数据标注出处 |
| 概率说明依据 | 每个概率数字写明推导逻辑 |
| 声明免责 | 输出结尾声明"仅供分析参考" |
| 信息冲突标注 | 多源不一致时明确列出 |
| 时效性标注 | 标注"截至 YYYY-MM-DD" |
