# 数据来源与核验规范（SOURCE_POLICY）

## 概述

本文件规定了使用本 Skill 时涉及实时数据的获取、引用、核验和冲突处理标准。
所有分析输出必须遵守本规范，确保信息来源可追溯、可验证。

---

## 一、数据来源优先级

按可靠度从高到低排列：

| 优先级 | 来源类型 | 示例 | 适用场景 |
|--------|----------|------|----------|
| P0 | 用户提供的确认数据 | 用户粘贴的官方公告 | 用户已自行核验 |
| P1 | 官方机构 | FIFA.com、各大洲足联、各国足协 | 积分榜、赛程、规则 |
| P2 | 球队官方渠道 | 俱乐部/国家队官网、官方社媒 | 伤病、首发、转会确认 |
| P3 | 权威体育媒体 | ESPN、BBC Sport、The Athletic、Goal | 赛事报道、分析 |
| P4 | 数据统计平台 | FBref、Transfermarkt、WhoScored、Sofascore | 技术统计、历史数据 |
| P5 | 中文体育媒体 | 懂球帝、直播吧、虎扑 | 中文报道、国内视角 |
| P6 | 模型已有知识 | AI 训练数据中的历史信息 | **仅作为补充，不可作为实时来源** |

---

## 二、引用格式规范

每次输出涉及实时或事实性数据时，**必须包含**以下元信息：

### 2.1 查询时间

```
📅 数据查询时间：YYYY-MM-DD HH:mm (时区)
```

示例：
```
📅 数据查询时间：2026-07-10 14:30 (UTC+8)
```

### 2.2 来源标注

行内引用格式：

```
【来源：FIFA.com，2026-07-10】
【来源：ESPN 报道，记者 XXX，2026-07-09】
【来源：球队官方 Twitter，2026-07-10】
```

表格/列表中的批量引用：

```
> 📊 数据来源：FBref (fbref.com)，统计截至 2026-07-09
```

### 2.3 数据状态标签

每条关键数据必须标注其确认程度：

| 标签 | 含义 | 使用场景 |
|------|------|----------|
| ✅ 官方确认 | 由官方机构或球队正式发布 | 首发名单、伤病公告、积分榜 |
| 📰 媒体报道 | 由权威媒体报道但未获官方确认 | 伤情更新、转会传闻、预计首发 |
| ⚠️ 未确认 | 来源可靠度不足或信息矛盾 | 小道消息、社媒爆料 |
| 🕐 时效待验 | 数据可能已过时，需重新核验 | 超过24小时的伤病/首发信息 |

---

## 三、数据核验执行规则

### 3.1 必须联网核验的信息

以下信息**禁止凭记忆输出**，必须实时查询：

- 最新比分与赛果
- 实时积分榜与排名
- 首发阵容与替补名单
- 最新伤病/停赛信息
- 赛程变动（时间/场地）
- 球员转会状态

### 3.2 可使用已有知识的信息

以下信息允许使用模型知识，但建议交叉验证：

- 历史交锋记录（需标注"根据历史数据"）
- 球队战术风格描述
- 教练执教理念
- 规则解释（FIFA 规则、VAR规则）

### 3.3 时效性要求

| 数据类型 | 有效窗口 | 超时处理 |
|----------|----------|----------|
| 比分/赛果 | 赛后立即 | 标注"赛后 X 小时数据" |
| 首发阵容 | 赛前 1 小时 | 标注"预计首发，待官方确认" |
| 伤病信息 | 24 小时内 | 超时标注 🕐 |
| 积分榜 | 本轮结束后 | 标注"截至第 X 轮" |
| 转会信息 | 当日 | 标注"截至 XX 日报道" |

## 四、冲突处理规则

当多个来源信息不一致时，按以下规则处理：

### 4.1 优先级规则

```
官方公告 > 权威媒体一致报道 > 单一媒体报道 > 社媒爆料 > 模型记忆
```

### 4.2 冲突标注格式

当信息存在冲突时，**必须在输出中明确说明**：

```
⚠️ 信息冲突提示：
- FIFA 官网显示：XX 球员在注册名单中（✅ 官方确认）
- ESPN 报道：该球员因伤缺席训练（📰 媒体报道，2026-07-09）
- 球队官方尚未发布伤病公告
- 结论：暂按"出战存疑"处理，建议关注赛前 1 小时官方首发公告
```

### 4.3 无法核验时的处理

如果联网查询失败或无法获取可靠信息：

1. **不编造数据填充**
2. 明确标注 `[数据暂缺，未能核验]`
3. 说明已尝试的查询途径
4. 给出基于有限信息的保守判断，并标注置信度

## 五、不确定性声明规范

所有分析结论必须包含不确定性说明：

### 5.1 预测类结论

```
🎯 预测结论：A 队小胜概率最高
📊 置信度：中等（65%）
⚠️ 不确定因素：
  - A 队关键球员伤情尚未官方确认
  - 双方历史交锋样本量偏小（仅 3 场）
  - 未考虑天气/场地等外部因素
```

### 5.2 事实类结论

```
📋 结论：A 队目前积 6 分排名小组第一
✅ 确认状态：官方确认（FIFA.com，截至第2轮）
⚠️ 注意：第3轮结果将改变排名，本结论有效期至末轮开赛前
```

### 5.3 推测类结论

```
💭 推测：教练可能采用 4-3-3 阵型
📰 依据：ESPN 赛前报道 + 近 3 场阵型趋势
⚠️ 可靠度：中低（基于媒体推测，非官方确认）
```

---

## 六、禁止行为

1. **禁止虚构比分、进球、数据**：如果不确定，写"暂无数据"而非编造
2. **禁止虚构来源**：不编造不存在的报道或文章
3. **禁止过期数据当作实时数据**：超过时效窗口的数据必须标注
4. **禁止将模型记忆当作事实**：知识截止日期后的信息必须联网核验
5. **禁止隐藏信息冲突**：发现矛盾必须如实暴露

---

## 七、合规检查清单

输出前自查：

- [ ] 所有实时数据是否标注了查询时间？
- [ ] 所有事实性陈述是否标注了来源？
- [ ] 是否标注了数据状态（官方确认/媒体报道/未确认）？
- [ ] 是否存在信息冲突？如有，是否已标注？
- [ ] 预测结论是否包含置信度和不确定因素？
- [ ] 是否存在过期数据？如有，是否已标注时效？
- [ ] 是否存在无法核验的信息？如有，是否已标注 `[数据暂缺]`？
