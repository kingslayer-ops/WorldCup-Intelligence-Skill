# 数据来源与核验规范

本文件规定使用本 Skill 时涉及数据的获取、引用、核验和冲突处理标准。

---

## 一、来源优先级

| 优先级 | 来源类型 | 示例 | 使用场景 |
|--------|----------|------|----------|
| S1 | 用户提供的一手数据 | 用户上传的积分榜、赛程、阵容 | 用户明确指定数据时优先 |
| S2 | 官方来源 | FIFA、洲际足联、国家队官网、足协公告 | 赛程、伤停、名单、纪律处罚 |
| S3 | 权威体育媒体 | BBC Sport、ESPN、The Athletic、Sky Sports | 新闻、赛前报道、伤病跟踪 |
| S4 | 数据平台 | FBref、Transfermarkt、WhoScored、SofaScore | 球员数据、球队数据、历史统计 |
| S5 | 中文体育媒体 | 懂球帝、直播吧、虎扑 | 中文报道、国内视角 |
| S6 | 历史资料 | 往届世界杯记录、公开百科资料 | 背景说明、历史交锋 |
| S7 | 模型已有知识 | 仅作背景补充 | **不可用于实时数据** |

---

## 二、强制联网核验场景

以下信息 **禁止凭记忆输出**，必须实时查询确认：

| 数据类型 | 原因 | 核验来源 |
|----------|------|----------|
| 最新赛程 | 可能变动 | FIFA.com |
| 比分与赛果 | 必须准确 | FIFA.com、权威媒体 |
| 实时积分榜 | 每轮变化 | FIFA.com |
| 首发阵容 | 赛前 1 小时确定 | 球队官方 |
| 伤病名单 | 随时变化 | 球队官方、权威媒体 |
| 红黄牌停赛 | 每场可能新增 | FIFA.com |
| 淘汰赛对阵 | 依赖小组赛结果 | FIFA.com |
| 球员转会状态 | 影响国家队征召 | 官方确认 |
| 国家队最新名单 | 每期更新 | 足协官方 |
| 当前主教练 | 可能换帅 | 官方公告 |
| 战术/阵型变化 | 依赖最近比赛 | 权威媒体分析 |

## 三、来源冲突处理

### 优先级规则

```
官方公告 > 多家权威媒体一致报道 > 单一权威媒体 > 中文媒体 > 社媒爆料 > 模型记忆
```

### 冲突处理流程

```
发现信息冲突
    ├── 有官方来源？ → 以官方为准，标注其他来源说法
    ├── 多家权威一致？ → 采用多数说法，标注少数不同意见
    ├── 各方说法不一？ → 全部列出，不选择性采信
    └── 均无法确认？ → 标注"未确认"，不编造
```

### 冲突标注格式

```markdown
⚠️ **信息冲突提示**：
- FIFA 官网：XX 球员在注册名单中（✅ 官方确认）
- ESPN 报道：该球员因伤缺席训练（📰 媒体报道，YYYY-MM-DD）
- 球队官方：尚未发布伤病公告
- **结论**：暂按"出战存疑"处理，建议关注赛前官方首发公告
```

### 无法核验时的处理

1. **不编造数据填充**
2. 明确标注 `[数据暂缺，未能核验]`
3. 说明已尝试的查询途径
4. 给出基于有限信息的保守判断，并标注置信度为"低"

## 四、输出引用要求

每次涉及实时数据时，输出应包含：

### 4.1 数据截止时间

```markdown
📅 数据截止时间：YYYY-MM-DD HH:mm (时区)
```

### 4.2 数据来源

```markdown
📰 数据来源：
- 积分榜：FIFA.com（✅ 官方确认）
- 伤病信息：ESPN 报道（📰 媒体报道）
- 首发预测：The Athletic（📰 媒体报道，未官方确认）
```

### 4.3 数据状态

| 标签 | 含义 | 使用场景 |
|------|------|----------|
| ✅ 官方确认 | 官方机构正式发布 | 赛程、积分榜、官方名单 |
| 📰 媒体报道 | 权威媒体报道但未获官方确认 | 预计首发、伤情更新 |
| 👤 用户提供 | 用户自行提供 | 用户指定的积分榜或假设数据 |
| ⚠️ 未确认 | 来源可靠度不足或信息矛盾 | 小道消息、社媒爆料 |
| 🕐 时效待验 | 数据可能过时 | 超过 24 小时的伤病信息 |

### 4.4 不确定信息说明

```markdown
⚠️ **不确定信息**：
- A 队 XX 球员伤情：仅 ESPN 单一来源报道，球队未确认
- B 队阵型：基于近 3 场推测，非官方确认
```

## 五、时效性要求

| 数据类型 | 有效窗口 | 超时处理 |
|----------|----------|----------|
| 比分/赛果 | 赛后即时 | 标注"赛后 X 小时数据" |
| 首发阵容 | 赛前 1 小时 | 标注"预计首发，待官方确认" |
| 伤病信息 | 24 小时内 | 超时加 🕐 标签 |
| 积分榜 | 本轮结束后 | 标注"截至第 X 轮" |
| 转会信息 | 当日 | 标注"截至 XX 日报道" |

---

## 六、禁止行为

| 禁止行为 | 说明 |
|----------|------|
| 编造实时数据 | 不确定时写"暂无数据" |
| 编造伤病名单 | 必须有来源支撑 |
| 编造首发阵容 | 未确认时标注"预计" |
| 编造出线概率 | 必须说明计算方法 |
| 编造来源 | 不编造不存在的报道或文章 |
| 过期数据当实时 | 超过时效窗口必须标注 |
| 把模型预测伪装成官方结论 | 必须声明"分析参考" |
| 提供投注建议 | 不推荐投注金额或方向 |
| 隐藏信息冲突 | 发现矛盾必须如实暴露 |

## 七、合规检查清单

输出前自查：

- [ ] 所有实时数据标注了截止时间？
- [ ] 所有事实性陈述标注了来源？
- [ ] 标注了数据状态（官方/媒体/未确认）？
- [ ] 信息冲突已标注？
- [ ] 预测结论包含依据和不确定因素？
- [ ] 不存在过期数据未标注的情况？
- [ ] 无法核验的信息已标注 `[数据暂缺]`？
- [ ] 未提供任何博彩/投注建议？
