# 分析工作流

本文件定义五大分析任务的标准执行流程。

---

## 一、赛前分析工作流

### 适用场景

比赛开始前 1–72 小时，为用户提供全面的赛前情报与预测分析。

### 必要输入

| 输入 | 说明 |
|------|------|
| 比赛双方 | 对阵的两支球队 |
| 比赛阶段 | 小组赛/16强/8强/半决赛/决赛 |
| 比赛日期 | 具体日期或"明天""后天" |
| 用户是否提供数据 | 若用户已提供数据则优先使用 |

### 可选输入

- 分析深度偏好（快速/标准/深度）
- 角色模式偏好（分析师/教练/解说员）
- 关注重点（战术/球员/数据/预测）

### 执行步骤

```
1. 核验赛程和比赛背景
   └── 确认对阵、时间、场地、赛事阶段

2. 核验双方近期状态
   └── 近 5 场战绩、进失球、士气、体能

3. 核验伤停和预计首发
   └── 伤病名单、停赛球员、预计阵容

4. 分析双方阵型和战术风格
   └── 常用阵型、进攻/防守体系、定位球

5. 建立战术对位矩阵
   └── 逐位置对比、突破口、弱侧

6. 提炼胜负手
   └── 决定比赛走向的 2-3 个关键因素

7. 给出预测倾向和不确定性
   └── 概率区间 + 依据 + 不确定因素

8. 输出爆冷风险
   └── 冷门触发因子 + 历史模式 + 风险等级
```

### 推荐输出结构

参照 `templates/赛前分析模板.md`

### 需要核验的数据

- 赛程和场地（FIFA.com）
- 近期战绩（数据平台）
- 伤病名单（官方/权威媒体）
- 历史交锋（数据平台）
- 预计首发（赛前 1 小时官方公布）

### 常见风险

| 风险 | 应对 |
|------|------|
| 伤病信息未确认 | 标注"📰 媒体报道，待官方确认" |
| 历史交锋样本小 | 降低历史因素权重并说明 |
| 首发未公布 | 标注"预计首发"而非确定 |
| 无近期直接交锋 | 使用间接参照（共同对手） |

## 二、赛后复盘工作流

### 适用场景

比赛结束后，提供结构化的赛后分析与战术复盘。

### 必要输入

| 输入 | 说明 |
|------|------|
| 比赛赛果 | 最终比分（含加时/点球） |
| 比赛双方 | 对阵球队 |
| 分析角度 | 战术为主/数据为主/叙事为主 |

### 可选输入

- 关注的特定球员
- 关注的特定时间段
- 是否需要假设推演

### 执行步骤

```
1. 确认赛果
   └── 最终比分、进球者、时间线、红黄牌

2. 获取技术统计
   └── 控球率、射门、传球、xG（如有）

3. 确认首发和换人
   └── 双方阵容、换人时间和人选

4. 识别关键转折点
   └── 进球、红牌、换人、VAR、受伤

5. 分析战术执行
   └── 实际阵型 vs 预期、战术调整、空间利用

6. 评估关键球员表现
   └── 评分、关键数据、攻防贡献

7. 分析教练决策
   └── 换人效果、临场调整、阵型变化

8. 评估后续影响
   └── 对积分榜/出线/下轮的影响
```

### 推荐输出结构

参照 `templates/赛后复盘模板.md`

### 需要核验的数据

- 最终比分和进球时间线
- 技术统计（FBref/WhoScored）
- 首发阵容和换人记录
- xG 数据（如有公开来源）

### 常见风险

| 风险 | 应对 |
|------|------|
| xG 数据不可得 | 标注"暂无 xG 数据"，使用射门数据替代 |
| 赛后即时数据可能修正 | 标注"赛后初步数据" |
| 球员评分主观性 | 引用第三方评分并标注来源 |

## 三、小组出线推演工作流

### 适用场景

小组赛进行中（通常第 2 轮结束后），推演各队出线概率。

### 必要输入

| 输入 | 说明 |
|------|------|
| 当前积分榜 | 各队积分、净胜球等（必须核验） |
| 剩余赛程 | 未踢比赛的对阵安排 |
| 排名规则 | FIFA 小组赛排名规则 |

### 可选输入

- 用户对各场比赛的主观概率
- 关注的特定球队
- 分析深度（简要/详细）

### 执行步骤

```
1. 获取并核验当前积分榜
   └── 必须联网确认，不可凭记忆

2. 确认剩余赛程
   └── 对阵安排和比赛时间

3. 明确 FIFA 排名规则
   └── 积分 → 净胜球 → 进球数 → 相互战绩 → 公平竞赛 → 抽签

4. 枚举所有可能的结果组合
   └── 每场比赛 3 种结果（胜/平/负）

5. 为每场比赛赋值概率
   └── 必须说明赋值方法和依据

6. 计算各情景下的最终积分
   └── 识别同分情况，应用排名规则

7. 汇总出线概率
   └── 使用区间表达（如 60%–75%）

8. 分析末轮博弈因素
   └── 动力/摆烂/默契球风险
```

### 推荐输出结构

参照 `templates/小组出线推演模板.md`

### 需要核验的数据

- 当前积分榜（必须联网核验）
- 剩余赛程（必须联网核验）
- 已踢比赛的具体比分（用于相互战绩判断）

### 常见风险

| 风险 | 应对 |
|------|------|
| 积分榜数据过时 | 标注"截至第 X 轮" |
| 净胜球情景复杂 | 说明简化处理或列出关键阈值 |
| 概率赋值主观 | 明确说明方法和依据 |
| 三队同分规则复杂 | 逐条应用 FIFA 规则 |

## 四、战术对位分析工作流

### 适用场景

需要深入分析双方阵型匹配和位置对比时使用。

### 必要输入

| 输入 | 说明 |
|------|------|
| 双方阵型 | 如 4-3-3 vs 3-4-2-1 |
| 分析目的 | 赛前预判/赛后复盘/教学演示 |

### 可选输入

- 具体球员名单
- 关注的特定区域
- 历史类似对位参考

### 执行步骤

```
1. 绘制双方阵型示意
   └── ASCII 或文字描述位置分布

2. 分析阵型匹配度
   └── 哪些区域人数对等/不对等

3. 逐区域对位分析
   └── 边路对位、中场对位、锋线对位

4. 识别突破口
   └── 人数优势区域、弱侧、空间

5. 分析攻防转换
   └── 高位压迫应对、反击路径

6. 预判变阵可能
   └── 领先/落后时的调整

7. 总结关键对位
   └── 2-3 个决定比赛的对位点
```

### 推荐输出结构

参照 `templates/赛前分析模板.md` 中的战术对位章节

### 需要核验的数据

- 双方近期使用的阵型（数据平台/权威媒体）
- 关键球员的位置热力图（如有）
- 教练的战术偏好变化

### 常见风险

| 风险 | 应对 |
|------|------|
| 阵型可能临时变阵 | 分析主阵型 + 讨论变阵可能 |
| 球员位置灵活 | 说明关键球员的多位置能力 |
| 缺少具体球员数据 | 使用阵型理论分析 |

## 五、内容生成工作流

### 适用场景

为多平台创作赛事相关内容（短视频/公众号/社媒）。

### 必要输入

| 输入 | 说明 |
|------|------|
| 内容主题 | 赛前预热/赛后复盘/话题讨论 |
| 目标平台 | 抖音/快手/公众号/微博/Twitter |
| 时长/字数 | 60秒/3分钟/2000字 等 |

### 可选输入

- 风格偏好（严肃/轻松/数据流/情感流）
- 目标受众
- 参考案例

### 执行步骤

```
1. 确认需求
   └── 平台、类型、风格、时长/字数

2. 准备素材
   └── 从分析中提取关键信息和数据点

3. 确定叙事主线
   └── 核心论点、悬念钩子、情绪节奏

4. 撰写内容
   └── 按平台格式和时长要求创作

5. 添加互动引导
   └── 提问、投票、评论引导

6. 审核合规
   └── 事实性、版权、博彩暗示、平台规范

7. 输出制作建议
   └── BGM、字幕、转场、封面文案
```

### 推荐输出结构

参照 `templates/内容脚本模板.md`

### 需要核验的数据

- 内容中引用的数据和事实
- 球员/球队的当前状态（如需要）

### 常见风险

| 风险 | 应对 |
|------|------|
| 引用数据不准确 | 标注来源或使用"约"表达 |
| 涉及版权素材 | 提醒用户注意素材版权 |
| 博彩暗示 | 严格禁止任何投注引导 |
| 敏感言论 | 避免极端表述和歧视性内容 |

## 通用质量标准

所有工作流执行完毕后，必须通过以下检查：

- [ ] 遵守 `SOURCE_POLICY.md` 数据核验规范
- [ ] 使用对应标准模板输出
- [ ] 预测/推测有依据支撑
- [ ] 不确定性已声明
- [ ] 避免绝对化表述
- [ ] 无博彩/投注建议
- [ ] 声明"仅供分析参考"
