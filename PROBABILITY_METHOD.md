# 概率与推演方法说明

本文件说明本 Skill 在进行比赛预测和出线推演时的概率表达规范、估算方法和局限性。

---

## 一、核心声明

> ⚠️ **免责声明**：概率输出仅代表基于输入条件的模型化推演，不等同于官方结论，不应用作博彩或投注依据。任何预测都存在不确定性，足球比赛的随机性决定了精确预测不可行。

---

## 二、概率表达规范

### 2.1 推荐表达方式

| 表达方式 | 适用场景 | 示例 |
|----------|----------|------|
| 概率区间 | 比赛胜负预测 | "A 队获胜概率约 45%–55%" |
| 定性描述 | 快速分析 | "A 队小胜概率较高" |
| 区间 + 定性 | 标准分析 | "A 队较大概率获胜（55%–65%），但存在中等爆冷风险" |

### 2.2 禁止表达方式

| 禁止 | 原因 |
|------|------|
| "A 队必胜" | 绝对化表述 |
| "出线概率 92%" | 无方法说明的伪精确数字 |
| "巴西将击败克罗地亚" | 把预测当确定事实 |
| "建议押 A 队" | 博彩建议 |
| "这场稳了" | 隐含确定性 |

### 2.3 精度规则

- **比赛预测**：使用 10% 宽度的区间（如 45%–55%），不给伪精确单点值
- **出线推演**：可使用 5%–10% 宽度区间（如 70%–80%）
- **爆冷风险**：使用定性等级（低/中/高），可附概率区间
- **若数据不足**：不给出概率，改用"目前信息不足以进行量化预测"

## 三、比赛预测方法

### 3.1 多维度加权分析

本 Skill 推荐的比赛预测思路：

```
最终预测 = f(近期状态, 历史交锋, 阵容完整度, 战术匹配, 场地因素, 赛事阶段)
```

| 维度 | 权重建议 | 数据来源 |
|------|----------|----------|
| 近期状态 | 高 | 近 5 场战绩、进失球 |
| 阵容完整度 | 高 | 伤停名单、关键球员缺阵 |
| 战术匹配 | 中 | 阵型克制关系 |
| 历史交锋 | 中（样本够时） | 历史战绩 |
| 场地/气候 | 低–中 | 主场优势、气候适应 |
| 赛事阶段 | 低–中 | 淘汰赛心态、动力因素 |

### 3.2 预测输出要求

每次给出预测时必须包含：

```markdown
## 预测倾向

- **倾向**：A 队小胜（概率区间：50%–60%）
- **依据**：
  1. [具体依据 1]
  2. [具体依据 2]
  3. [具体依据 3]
- **不确定因素**：
  - [因素 1]
  - [因素 2]
- **爆冷风险**：中等
- ⚠️ 以上为分析倾向，非确定结论，不构成任何投注建议
```

## 四、小组出线推演方法

### 4.1 情景枚举法（推荐）

**适用条件**：剩余比赛不超过 4 场（即最多 3^4 = 81 种组合）

**步骤**：

```
Step 1：输入当前积分榜
    └── 必须联网核验或由用户提供

Step 2：输入剩余赛程
    └── 确认所有未踢比赛

Step 3：为每场比赛设置胜/平/负概率
    └── 必须说明赋值方法（见 4.3）

Step 4：枚举所有可能的结果组合
    └── N 场比赛 = 3^N 种组合

Step 5：计算每种组合下各队最终积分
    └── 当前积分 + 新增积分

Step 6：按 FIFA 排名规则排序
    └── 处理同分情况（见 4.2）

Step 7：统计每队出线次数
    └── 出线次数 / 总情景数 × 情景概率

Step 8：输出概率区间
    └── 考虑概率赋值的不确定性，使用区间表达
```

### 4.2 FIFA 小组赛排名规则

当两队或多队积分相同时，按以下顺序决定排名：

| 优先级 | 规则 | 说明 |
|--------|------|------|
| 1 | 积分 | 总积分（胜3/平1/负0） |
| 2 | 净胜球 | 总进球 - 总失球 |
| 3 | 进球数 | 总进球数 |
| 4 | 相互战绩积分 | 同分球队之间的比赛积分 |
| 5 | 相互战绩净胜球 | 同分球队之间的净胜球 |
| 6 | 相互战绩进球数 | 同分球队之间的进球数 |
| 7 | 公平竞赛积分 | 红黄牌罚分（黄牌-1，红牌-3等） |
| 8 | 抽签 | FIFA 组委会抽签决定 |

> ⚠️ 注意：不同届世界杯的同分排名规则可能略有调整，使用前应核验当届赛事的官方规则。2026 世界杯采用 48 队 12 组赛制，排名规则可能与往届不同。

### 4.3 概率赋值方法

为每场比赛赋值胜/平/负概率时，**必须选择并声明方法**：

| 方法 | 适用场景 | 优点 | 局限 |
|------|----------|------|------|
| 实力差距估算 | 信息有限时 | 简单快速 | 主观性强 |
| 历史基准率 | 有足够类似比赛参考 | 有数据支撑 | 样本可能不代表当前 |
| Elo/FIFA 排名换算 | 排名差距明确 | 量化客观 | 不反映近期状态 |
| 蒙特卡洛模拟 | 需要精确推演 | 覆盖复杂情景 | 依赖输入假设 |
| 博彩赔率参考 | 需要市场共识 | 聚合大量信息 | 含庄家利润，非纯概率 |

**输出时必须声明**：

```markdown
📊 **概率赋值方法**：[所选方法]
📋 **赋值依据**：[具体说明为什么这样赋值]
⚠️ **方法局限**：[已知局限性]
```

### 4.4 简化处理说明

当情景过多或净胜球情况复杂时，允许简化：

- 合并极低概率情景（< 1%）
- 净胜球采用"当前差距 + 预期进球"估算
- 标注"简化处理，精确值需蒙特卡洛模拟"

## 五、不确定性来源

所有概率估算都受以下因素影响：

| 不确定性来源 | 影响程度 | 应对方式 |
|--------------|----------|----------|
| 概率赋值的主观性 | 高 | 使用区间而非单点值 |
| 未知伤病/状态变化 | 中–高 | 标注信息截止时间 |
| 比赛随机性 | 高 | 不做精确预测 |
| 心理和动力因素 | 中 | 定性讨论而非量化 |
| 裁判/VAR 影响 | 低–中 | 不纳入概率计算 |
| 天气/场地 | 低 | 仅作定性补充 |
